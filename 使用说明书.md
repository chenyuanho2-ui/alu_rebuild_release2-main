# 铝材加热控制系统使用说明书

## 一、系统概述

本系统是一款基于STM32H750微控制器的铝材激光加热控制系统，集成TouchGFX触摸屏界面，支持多种PID控温算法，具备SD卡数据记录、RS485通信等功能。

### 1.1 硬件配置

| 组件 | 型号/说明 |
|------|----------|
| 主控芯片 | STM32H750XB |
| 显示屏 | 800x480 TFT LCD (TouchGFX) |
| 温度传感器 | ADS1118双通道热电偶采集 |
| 通信接口 | RS485、UART |
| 存储 | SD卡 (FatFS文件系统) |
| 控制算法 | 标准PID / 模糊PID / 高级PID |

### 1.2 软件架构

- **操作系统**: FreeRTOS
- **GUI框架**: TouchGFX
- **文件系统**: FatFS

---

## 二、界面说明

### 2.1 主界面 (Screen 0)

主界面显示当前温度、功率阈值设置，以及SD卡文件列表。

**显示区域**:
- 温度阈值显示区 (Temp Area)
- 功率阈值显示区 (Power Area)
- 文件列表滚动区

**阈值选择高亮规则**:
- 青色高亮 = 当前选中可调
- 白色 = 非选中状态

### 2.2 加热监控界面 (Screen 1)

脚踏触发后自动切换至此界面，实时显示温度曲线和PID参数变化。

---

## 三、按键操作

系统共5个按键：4个面板按键 + 1个脚踏开关

### 3.1 按键功能表

| 按键 | 单独按下 | 与Key1组合 |
|------|---------|-----------|
| Key 1 | 切换温度/功率设置选择 | - |
| Key 2 | 增加当前阈值 | Key1+Key4: 温度修正开关 |
| Key 3 | 减少当前阈值 | - |
| Key 4 | 发送RS485功率设置指令 | - |
| 脚踏 | 启动加热/停止加热 | - |

### 3.2 阈值设置范围

| 参数 | 最小值 | 最大值 | 步进 |
|------|-------|-------|------|
| 温度阈值 | 0°C | 150°C | 5°C |
| 功率阈值 | 0.0 | 9.0 | 0.1 |

### 3.3 脚踏操作

**启动加热**: 在非加热状态下踩下脚踏
- 系统自动创建新的CSV数据文件
- 自动切换到加热监控界面
- PID闭环控温启动

**停止加热**: 再次踩下脚踏
- 发送关闭激光指令
- 保存数据到SD卡
- 返回主界面

### 3.4 温度修正功能

同时按下 **Key1 + Key4** 可开启/关闭温度修正模式：
- LED熄灭 = 温度修正关闭
- LED点亮 = 温度修正开启
- 修正值默认 +10°C

---

## 四、串口指令

系统支持通过串口进行PID参数调节和激光测试。

### 4.1 指令列表

| 指令 | 功能 |
|------|------|
| `help` | 显示所有可用指令 |
| `set_pid` | 进入PID参数调节模式 |
| `laseron` | 进入激光测试模式 |
| `laseroff` | 退出激光测试模式 |

### 4.2 PID调节流程

```
1. 发送: set_pid
2. 系统回复: "PID Tuning Mode ON" 及当前参数
3. 发送新参数 (格式: Kp,Ki,Kd，例如: 40,0.8,125)
4. 系统解析后确认: "Confirm [Y], Re-enter [N]:"
5. 发送 Y 确认或 N 重新输入
```

### 4.3 激光测试流程

```
1. 发送: laseron
2. 系统回复: "Enter target current and PWM (format: X,Y)"
3. 发送目标值 (例如: 2.5,500)
4. 系统确认: "Confirm apply? [Y/N]:"
5. 发送 Y 确认
6. 踩下脚踏即可测试激光
7. 发送 laseroff 退出测试模式
```

### 4.4 注意事项

- PID调节功能需 `enable_pid_tune=1` 且系统处于非加热状态
- 激光测试功能需 `enable_laser_test=1` 且系统处于非加热状态

---

## 五、PID算法说明

系统支持三种PID算法，通过 `pid_algorithm_type` 变量切换：

| 算法类型值 | 算法名称 | 说明 |
|-----------|---------|------|
| 0 | 标准PID | 经典比例-积分-微分控制 |
| 1 | 模糊PID | 结合模糊逻辑的自适应PID |
| 2 | 高级PID | 改进型PID控制器 |

### 5.1 参数保存

- 标准PID参数保存到SD卡配置文件
- 模糊PID和高级PID参数在运行时覆盖

## 六、数据记录

### 6.1 文件格式

系统自动创建CSV格式文件，文件名格式: `data_N.csv`

**数据列说明**:

| 列序 | 内容 | 示例 |
|------|------|------|
| 1 | 序号 | 1, 2, 3... |
| 2 | 实时温度(°C) | 85.50 |
| 3 | 比例项输出 | 0.123 |
| 4 | 积分项输出 | 0.045 |
| 5 | 微分项输出 | 0.067 |

**表头**: `index,temperature,speed_p,speed_i,speed_d`

### 6.2 记录控制

- 加热启动时自动创建新文件
- 加热停止时自动保存
- 通过 `sd_record_enable` 变量可禁用记录

### 8.4 SD卡配置文件

| 文件名 | 用途 | 格式 |
|-------|------|------|
| `pid_cfg.txt` | 保存PID参数 | `Kp,Ki,Kd` (例如: 40.00,0.80,125.00) |
| `data_N.csv` | 加热数据记录 | CSV格式，见第六章 |

### 8.5 修改建议

- **运行时调整**: 通过串口发送 `set_pid` 指令(需 `enable_pid_tune=1`)
- **代码修改**: 直接修改对应变量的初始化值
- **持久化**: 标准PID参数会保存到SD卡，重启后自动加载

## 十、系统架构与外设说明

### 10.1 系统架构概述

本激光热止血针嵌入式系统以STM32 Cortex-H7为核心构建分层代码架构：

```
┌─────────────────────────────────────────────────────────────┐
│                     应用层 (Application)                     │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐     │
│  │  TouchGFX GUI │  │  FreeRTOS    │  │  数据存储    │     │
│  │  界面交互    │  │  任务调度    │  │  SD卡记录    │     │
│  └──────────────┘  └──────────────┘  └──────────────┘     │
├─────────────────────────────────────────────────────────────┤
│                     中间层 (Middleware)                      │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐     │
│  │  PID控制     │  │  温度滤波    │  │  文件系统    │     │
│  │  算法库      │  │  双通道采集  │  │  FatFS       │     │
│  └──────────────┘  └──────────────┘  └──────────────┘     │
├─────────────────────────────────────────────────────────────┤
│                     驱动层 (HAL/CMSIS)                      │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────┐│
│  │SPI/UART│ │PWM/TIM │ │LTDC/IIC│ │FMC/QSPI│ │SDMMC ││
│  └─────────┘ └─────────┘ └─────────┘ └─────────┘ └─────┘│
├─────────────────────────────────────────────────────────────┤
│                     硬件层 (STM32H750)                      │
└─────────────────────────────────────────────────────────────┘
```

### 10.2 软件技术栈

| 层次 | 技术/组件 | 说明 |
|------|----------|------|
| 硬件抽象 | CMSIS_v1 | ARM Cortex微控制器软件接口标准 |
| 操作系统 | FreeRTOS | 实时多任务调度 |
| GUI框架 | TouchGFX | 轻量级嵌入式图形库 |
| 文件系统 | FatFS | SD卡文件系统 |
| 外设驱动 | HAL/LL | STM32硬件抽象层 |

### 10.3 外设接口映射

| 外设 | 接口 | 功能 |
|------|------|------|
| 温度传感器 | SPI2 | ADS1118双通道热电偶采集 |
| 激光通信 | UART4 + RS485 | 功率指令发送 |
| 激光控制 | TIM1_CH4 (PWM) | PWM占空比调节 |
| 显示屏 | LTDC + I2C | 800x480 TFT驱动 + 触控 |
| 数据存储 | SDMMC1 | SD卡读写 |
| 调试串口 | USART1 | 串口指令交互 |
| DAC输出 | DAC1_CH1 | 激光电流控制 |

### 10.4 内存扩展接口

| 存储类型 | 接口 | 容量 |
|---------|------|------|
| SDRAM | FMC | 32MB (0xC0000000) |
| Nor Flash | QSPI | 扩展存储 |
| SD卡 | SDMMC | 可移动存储 |

### 10.5 系统任务调度

FreeRTOS管理以下并发任务：

| 任务名称 | 优先级 | 周期 | 功能 |
|---------|-------|------|------|
| aluMain | Normal | 50ms | 按键扫描、UI更新 |
| Task_Control | High | 10ms | PID闭环控温 |
| aluSubProgress | Low | - | 加热进度显示 |
| SD卡写入 | - | 异步 | CSV数据持久化 |

## 十一、安全注意事项

1. **温度限制**: 最高设置温度不超过150°C
2. **功率限制**: 最大功率等级不超过9.0
3. **散热要求**: 设备应有适当散热措施
4. **紧急停止**: 踩下脚踏即可立即停止加热
5. **状态检查**: 加热前确认温度传感器工作正常

---

## 十二、故障排除

| 现象 | 可能原因 | 解决方法 |
|------|---------|---------|
| 温度显示异常 | 热电偶连接不良 | 检查SPI接线 |
| SD卡文件未创建 | FatFs挂载失败 | 检查SD卡初始化 |
| 串口无响应 | 波特率不匹配 | 确认UART4波特率设置 |
| 脚踏无反应 | 系统处于加热状态 | 先停止当前加热 |
| PID不收敛 | 参数设置不当 | 通过串口调节PID参数 |
